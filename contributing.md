# Contributing

Add only high-quality Native SDK resources.

## Criteria

- The resource must be directly about Native SDK or a working app built with Native SDK.
- The resource must be public, documented, and useful to Native SDK developers.
- The resource must not be archived, abandoned, deprecated, or mostly empty.
- Apps should include enough README detail to understand how Native SDK is used.
- Prefer links that end in `#readme` for GitHub repositories.

## Formatting

- Keep entries alphabetized within a section when there is no stronger reason to group them otherwise.
- Use this format: `- [Name](https://example.com#readme) - Description.`
- Start descriptions with an uppercase letter and end with a period.
- Keep descriptions objective. Do not use marketing copy.
- Do not add duplicate entries.

## Pull Requests

- Explain why the resource belongs on the list.
- Add one resource per pull request when possible.
- Run `bunx awesome-lint` before submitting, if available.
